# 🚀 Project Overview

**AI GitHub Engineer** is an AI-powered software engineering platform built using **Python**, **Streamlit**, the **GitHub REST API**, **Ollama**, and **OpenAI**.

The application helps developers analyze GitHub repositories, review source code, generate documentation, perform security analysis, review pull requests, analyze commits, generate tests, evaluate repository quality, and interact with repositories using natural language.

The platform supports both **local AI models through Ollama** and **cloud models through OpenAI**, allowing developers to choose the provider that best fits their workflow.

The project follows a modular architecture, making it easy to maintain, extend, and enhance with additional AI engineering capabilities.

# 🏆 Five-Minute Introduction

Hello everyone.

Today I'd like to present my project called **AI GitHub Engineer**, which is one of my largest software engineering projects.

The motivation behind this project came from observing that developers often switch between multiple tools for repository analysis, documentation, code reviews, pull requests, commit analysis, testing, and security checks.

I wanted to bring these capabilities together into a single AI-powered application.

The application is built using Python as the primary programming language and Streamlit for the web interface.

It connects to GitHub using the GitHub REST API to retrieve repository information including repository metadata, commits, branches, contributors, pull requests, and other project details.

For artificial intelligence, the application supports two providers.

The first is Ollama, which allows local execution of language models such as qwen2.5, Llama, Gemma, Mistral, and DeepSeek-Coder.

The second is OpenAI, which provides cloud-based models for more advanced reasoning.

The modular provider architecture allows switching between providers without changing the rest of the application.

The project is organized into multiple layers.

The UI layer contains all Streamlit pages.

The API layer communicates with GitHub.

The AI layer contains reusable engineering modules.

The utility layer handles configuration, sessions, logging, and helper functions.

Some of the major AI features include repository analysis, code review, code explanation, repository scoring, documentation generation, bug detection, security analysis, Docker analysis, workflow analysis, license analysis, repository chat, pull request review, commit analysis, issue generation, automated test generation, and project generation.

I also focused heavily on maintainability by separating the application into reusable modules instead of building a monolithic application.

During development, I learned how to design reusable software components, work with REST APIs, integrate multiple AI providers, engineer effective prompts, organize larger Python projects, and create production-oriented applications.

This project represents my interest in combining artificial intelligence with practical software engineering workflows and serves as one of my primary portfolio projects.

# 📝 Detailed Project Explanation

AI GitHub Engineer is an AI-assisted software engineering platform designed to help developers understand, evaluate, and improve GitHub repositories.

Instead of using separate tools for documentation, code review, testing, pull request analysis, repository scoring, and developer assistance, this application combines these capabilities into one platform.

The system retrieves repository information from GitHub, processes the data, constructs structured prompts, sends them to either OpenAI or Ollama, and presents well-formatted engineering reports through a Streamlit interface.

Its modular design makes it easier to maintain, extend, and support additional AI providers and engineering tools in future releases.
